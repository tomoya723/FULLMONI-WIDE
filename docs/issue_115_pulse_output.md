# Issue #115: タコ・車速パルス出力機能（EPS駆動用）

## 概要

外付け電動パワーステアリング（EPS）ユニットへの駆動信号として、CAN受信済みのエンジン回転数（RPM）および車速（km/h）をパルス信号として出力する機能を追加する。

JIS D 5601:1992 に準拠したオープンコレクタ矩形波出力をトランジスタバッファ経由で供給する。

---

## ハードウェア構成

### 出力ピン割り当て

| 役割 | MCUピン | DOチャンネル | 外部バッファ | 外部コネクタ | 出力電圧 |
|------|---------|------------|------------|------------|--------|
| タコパルス出力 | PC2 | DO8 | Q5（NPN シングル） | DO2-0 | 12V系（外部プルアップ） |
| 車速パルス出力 | PC3 | DO9 | Q6（NPN シングル） | DO2-1 | 12V系（外部プルアップ） |

- Q5/Q6 はオープンコレクタ構成 → JIS D 5601 のオープンコレクタ仕様を満足
- 外部コネクタ（DO2-0/DO2-1）にて EPS 側 12V プルアップと直結可能
- PC2/PC3 は Config_PORT.c にて GPIO 出力として初期化済み（変更不要）

### PC4/PC5 の予約

- PC4(DO10) = MTIOC4A、PC5(DO11) = MTIOC4B に対応
- 将来 Hardware MTU4 PWM 昇格時の予備ピンとして保留

---

## パルス仕様

### タコパルス（JIS D 5601:1992 準拠）

| パラメータ | 値 |
|----------|----|
| パルス数 | 2 パルス / rev |
| 上限回転数 | 5000 rpm（5000 rpm 超は 5000 rpm 固定） |
| 最大出力周波数 | 5000 × 2 / 60 ≈ **166.7 Hz** |
| アイドル例（750 rpm） | 750 × 2 / 60 = **25.0 Hz** |
| RPM = 0 時 | 出力 LOW（パルス停止） |

```
f_tacho [Hz] = RPM × 2 / 60
```

### 車速パルス（JIS D 5601:1992 準拠）

| パラメータ | 値 |
|----------|----|
| パルス数 | PULSE_PER_KM = 2548 パルス / km |
| 最大入力速度 | 200 km/h |
| 最大出力周波数 | 200 × 2548 / 3600 ≈ **141.6 Hz** |
| 60 km/h 例 | 60 × 2548 / 3600 ≈ **42.5 Hz** |
| 速度 = 0 時 | 出力 LOW（パルス停止） |

```
f_speed [Hz] = speed_kmh × PULSE_PER_KM / 3600
```

## ソフトウェア設計

### タイマー設計

FIT CMT API（`r_cmt_rx`）を使用し、10 kHz（100 µs 周期）の割り込みを追加する。

| 項目 | 値 |
|-----|---|
| API | `R_CMT_CreatePeriodic(10000, pulse_out_isr, &handle)` |
| ベースクロック | PCLKA = 120 MHz |
| 割り込み周期 | 100 µs（10 kHz） |
| CMTチャンネル | 自動割り当て（emWin が CMT0 使用 → CMT1 が確保される） |
| Smart Configurator 変更 | **不要** |

**LCD 描画への影響**

- GLCDC + DMA によるピクセル転送は CPU 非依存のため無影響
- 10 kHz 割り込みは 30fps フレームあたり 333 回
- ISR 内処理はカウント比較＋ポートトグルのみ（数十サイクル）
- CPU 負荷 ≈ 0.2%（240 MHz 動作時）→ **実質無影響**

### カウンタ設計

ソフトウェア方形波生成：割り込みごとにカウンタをインクリメントし、ハーフ周期カウント値に達したらピンをトグルしてカウンタをリセットする。

```
half_count_tacho = 150,000 / RPM          [uint32_t]
half_count_speed = 18,000,000 / (speed × PULSE_PER_KM / 3600)
                 = 18,000,000 × 3600 / (speed × 2548)  [uint32_t]
```

| 条件 | half_count |
|------|-----------|
| RPM = 5000 rpm（最大） | 30 |
| RPM = 750 rpm（アイドル） | 200 |
| RPM = 100 rpm（極低速） | 1,500 |
| 200 km/h（最大） | 128 |
| 60 km/h | 426 |
| 1 km/h | 25,548 |

> `uint32_t` で保持することで 1 km/h（25,548 counts）も正確に処理。

### カウント誤差

- 10 kHz ベースで 5000 rpm 時は half = 30 → 周波数誤差 ±3.3%
- JIS D 5601 は信号精度について厳密規定なし（EPS ユニット側でアベレージング処理）
- 将来的に精度要件が判明した場合は PC4/PC5 + MTU4 ハードウェア PWM に昇格可能

## 定数定義

| マクロ | 値 | 意味 |
|-------|---|------|
| `TACHO_PULSE_PER_REV` | 2 | タコ 2 パルス/rev（JIS D 5601） |
| `TACHO_MAX_RPM` | 5000 | 上限クリップ |
| `PULSE_OUT_TIMER_HZ` | 10000 | ベースタイマー周波数（10 kHz） |
| `TACHO_HALF_FACTOR` | 150000 | = PULSE_OUT_TIMER_HZ × 60 / (TACHO_PULSE_PER_REV × 2) |
| `SPEED_HALF_FACTOR` | 18000000UL | = PULSE_OUT_TIMER_HZ × 3600 × 2 / 2 |

## 検証手順

1. ロジアナ/オシロで DO2-0（タコ）・DO2-1（車速）の周波数確認
   - CAN で RPM = 3000 を入力 → 期待値: `3000 × 2 / 60 = 100.0 Hz`
   - CAN で 60 km/h を入力 → 期待値: `60 × 2548 / 3600 ≈ 42.5 Hz`
   - CAN で 5001 rpm を入力 → クリップされ `166.7 Hz` に固定されることを確認
2. RPM = 0、速度 = 0 で出力が LOW に落ちることを確認
3. `debug_env/` の CAN シミュレーター（nodesim）でスイープ入力し、周波数追従を確認
