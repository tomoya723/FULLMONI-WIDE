FULLMONI-WIDE EEZ 参照関係マップ

src/ (共用コード) ↔ eez/eezXXX/ (デザイン別コード) 依存関係図

共用コード (src/) — 全デザインで同一のソースを使用

main.c
システム起動・メインループ。ペリフェラル初期化後に ui_dashboard_create() で画面生成し、 メインループ内で毎フレーム ui_dashboard_update() を呼んで表示更新。 USB CDC のモード遷移時に set_notify / clear_notify で通知オーバーレイを制御。
dataregister.c
CAN バスから受信した生データを物理値に変換・蓄積するデータ処理層。 g_CALC_data / g_CALC_data_sm 構造体にリアルタイム値を格納。 エラー検出時に set_notify で警告オーバーレイを表示。

シンボリックリンク層 (src/ui/, src/ui_binding/) — アクティブデザインの実体を参照

ui_dashboard.h
UI バインディングの公開インターフェース。4 関数のみ公開し、共用コード (main.c, dataregister.c) との契約を定義。 全デザインで同一シグネチャを保証する抽象化レイヤー。
ui_dashboard.c
デザイン別の実装本体。g_CALC_data の各フィールドを読み取り、LVGL ウィジェットに値をバインド。 タコメーター駆動、センサー値表示、バーゲージ更新、警告アイコン制御、テルテール演出などを実行。
screens.h
EEZ Studio が自動生成。objects_t 構造体で全 LVGL ウィジェットのポインタを保持。 デザインごとに構造体メンバーが異なる（eez003 は水平バー用 obj0-112 等の独自メンバーを持つ）。
eez_compat.h
ウィジェット名の変換マクロ。旧 SquareLine Studio 形式の名前 (ui_LblRPM 等) を EEZ Studio 形式 (objects.ui_lbl_rpm) にマップ。 デザインに存在しないウィジェットは NULL にマップし、使用側で安全にスキップ。
vars.h
EEZ Studio が自動生成するネイティブ変数アクセサ宣言。 EEZ Flow ランタイムが呼び出す get_var_rpm() / set_var_rpm() を宣言。 実装は各デザインの ui_dashboard.c 内に配置。

デザイン別コード (eez/eezXXX/) — switch_design.ps1 で切替

eez001 (Standard / 青テーマ)
円形アナログ針タコメーター + Arc ゲージ。6 本の lv_bar でセンサー値を表示。 時計・RPM 数値ラベル・AFR 等すべてのラベルを持つフル装備レイアウト。
eez002 (Racing / 赤テーマ)
eez001 と同一のウィジェット構成。配色・フォント・背景画像がレーシングテーマに変更。 screens.h の objects_t 構造体は eez001 と完全一致。
eez003 (Prodrive)
水平セグメントバー方式のタコメーター (obj0-24 = 25 本、各 360 RPM 幅)。 6 ゲージも水平バー化 (obj25-112)。針タコ・Arc・従来の lv_bar は廃止し、 ラベル名も独自 (charge_temp / boost_val / range)。 eez_compat.h で廃止ウィジェットを NULL にマップし、ui_dashboard.c 内で NULL ガード付きで処理。
